GOOGLE CALENDAR COPILOT LITE - BUILD STEPS (BEGINNER FRIENDLY)
Version: v1.0
Date: 2026-02-22

Goal: Follow these steps in Claude Code to build and test the app quickly.

==================================================
PHASE 0 - PREP (10-20 min)
==================================================
1) Install tools
- Node 20+
- npm
- Git
- Vercel CLI (optional for deploy)

2) Decide project folder
Example:
- ~/dev/Google-Calendar-AI-Manager/app

3) Prepare integration account
- Have Composio/Rube connection to Google Calendar ready
- Confirm connection is ACTIVE before app build tests

==================================================
PHASE 1 - SCAFFOLD PROJECT (10 min)
==================================================
Run:

mkdir -p ~/dev/Google-Calendar-AI-Manager/app
cd ~/dev/Google-Calendar-AI-Manager/app
npx create-next-app@latest . --ts --tailwind --eslint --app --src-dir --import-alias "@/*"

Then:
- npm run dev
- Confirm app opens at http://localhost:3000

==================================================
PHASE 2 - ASK CLAUDE CODE TO IMPLEMENT BASE STRUCTURE
==================================================
Paste this prompt into Claude Code:

"Build the base architecture for a simple Calendar Copilot Lite using Next.js App Router.
Create:
- src/app/page.tsx (dashboard shell)
- src/app/api/health/route.ts
- src/app/api/connection/route.ts
- src/lib/calendar/provider.ts (interface)
- src/lib/calendar/composioProvider.ts (stub implementation)
- src/lib/types.ts (EventSummary, ActionPreview, AuditEntry, UserSettings)
- src/components for StatusCard, EventList, FreeSlotForm, ActionPreviewModal, AuditLog
Use TypeScript strictly. No secrets in client code."

Verify:
- App compiles with no TS errors
- /api/health returns { ok: true }

==================================================
PHASE 3 - ENV + PROVIDER INTEGRATION
==================================================
Create .env.local:

COMPOSIO_API_KEY=YOUR_KEY
COMPOSIO_ACCOUNT_ID=YOUR_CONNECTED_ACCOUNT
DEFAULT_TIMEZONE=America/New_York
APP_WRITE_CONFIRMATION_REQUIRED=true

Paste this prompt into Claude Code:

"Implement server-side provider integration in composioProvider.ts and connect it to API routes.
Add routes:
- POST /api/events/list
- POST /api/events/find-free-slots
- POST /api/events/create-preview
- POST /api/events/create-confirm
- POST /api/events/update-preview
- POST /api/events/update-confirm
- POST /api/events/conflicts
Use preview-first writes and validation."

Verification:
- Read-only routes work first
- If write route is called without preview token/state, reject

PHASE 5 - GUARDRAILS + ERROR HANDLING
==================================================
Paste this prompt into Claude Code:

"Add strict safety controls:
- Visible timezone everywhere

==================================================
PHASE 6 - TESTING CHECKLIST (MANUAL)
==================================================
Run through this exact test matrix:

1. Connection
- Expected: connected status shown

2. List events
- Expected: upcoming events render correctly

3. Free slots
- Expected: 3-5 valid slots appear in selected range

4. Create preview
- Expected: diff shown before write

5. Create confirm
- Expected: event appears in calendar once

6. Update preview + confirm
- Expected: exact selected event is modified

7. Conflict scan
- Expected: known overlaps detected

8. Error handling
- Expected: revoked token shows clear re-auth message
